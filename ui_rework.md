# План глобальной переработки UI/UX Isolate Panel

На основе лучших решений из **metacubexd** (современный дизайн, карточки, glassmorphism, графики), **3x-ui** и **s-ui** (плотность данных, управление подписками и клиентами, наглядность статистики), составлен подробный план редизайна фронтенда панели. 

Главная цель: сделать интерфейс красивым, интуитивно понятным, отзывчивым (mobile-first) и информативным, сохранив высокую производительность Preact.

---

## 🎨 1. Дизайн-система и Базовые компоненты (Core UI)

Перед изменением страниц необходимо заложить прочный фундамент переиспользуемых UI-компонентов.

- [ ] **Цветовая палитра и Тематика (Tailwind)**
  - Улучшить темную тему (переход от абсолютного черного к глубоким оттенкам `zinc-950` или `slate-900`).
  - Настроить семантические цвета: `primary` (индиго/синий для акцентов), `success` (мягкий зеленый для онлайна/статуса "работает"), `warning` (оранжевый/желтый для истекающих сроков), `danger` (красный для ошибок/оффлайна).
  - Настроить эффекты "стекла" (Glassmorphism): полупрозрачные фоны с `backdrop-blur` для шапок, выпадающих меню и модальных окон.
- [ ] **Базовые UI компоненты (в `src/components/ui`)**
  - **Button:** Добавить варианты (`solid`, `outline`, `ghost`, `danger`) и размеры (`sm`, `md`, `lg`). Плавные `hover:scale` и `active:scale` эффекты.
  - **Card:** Карточки с мягкими тенями (`shadow-sm`, `shadow-md`), скруглениями (`rounded-xl` или `rounded-2xl`) и легким бордером (`border-white/5` для темной темы).
  - **Badge:** Компактные ярлычки для статусов (🟢 Active, 🔴 Stopped, 🟡 Expiring).
  - **Switch (Toggle):** Замена стандартных чекбоксов на iOS-подобные переключатели с плавной анимацией шарика.
  - **Progress Bar:** Горизонтальные и круговые прогресс-бары (особенно для отображения расхода трафика).
  - **Skeleton:** Заглушки для загружающегося контента (пульсирующие серые блоки), чтобы избежать "прыжков" верстки.
  - **Dropdown Menu:** Красивые выпадающие меню для действий (редактировать, удалить) с использованием `tippy.js` или кастомных порталов.
  - **Drawer / Slide-over:** Боковые всплывающие панели для редактирования/создания сущностей (вместо перехода на отдельную страницу).

## 📱 2. Глобальный Макет (Layout & Navigation)

Отказ от статичных и громоздких меню в пользу современных адаптивных решений.

- [ ] **Сайдбар (Sidebar)**
  - **Desktop:** Фиксированный слева, иконки + текст. Подсветка активного пункта градиентом или легким фоном.
  - **Mobile:** Скрыт по умолчанию. Открывается плавно поверх контента (Drawer) по кнопке "Гамбургер".
  - **Мини-статус внизу:** Отображение текущего ядра, версии панели и быстрая кнопка Logout.
- [ ] **Шапка (Header)**
  - "Хлебные крошки" (Breadcrumbs) для понимания текущего раздела (например: *Главная / Инбаунды / vless-tcp*).
  - Глобальные метрики в реальном времени (входящая/исходящая скорость стрелочками `↓ 1.2 MB/s  ↑ 4.5 MB/s`).
  - Переключатель Темной/Светлой темы и Языка.
- [ ] **Контентная область**
  - Ограничение максимальной ширины на сверхшироких мониторах (`max-w-7xl`).
  - Плавное появление страниц (Page Transitions) при роутинге.

## 📊 3. Дашборд (Главная страница)

Лицо панели. Пользователь должен сразу видеть состояние системы и сети.

- [ ] **Карточки системных ресурсов (System Stats)**
  - CPU: Кольцевой индикатор (Circle Progress) нагрузки.
  - RAM: Кольцевой или линейный индикатор использования памяти.
  - Uptime системы и сервисов (Sing-box/Xray).
- [ ] **Графики трафика (Traffic Charts)**
  - Красивый график (Area Chart) In/Out трафика за последние 24 часа или в реальном времени (потребует легковесную библиотеку, например Recharts, или CSS+SVG реализацию).
- [ ] **Сводка по подключениям**
  - Блок с общим количеством пользователей (Активно / Всего).
  - Блок с количеством активных соединений прямо сейчас.
  - Журнал последних событий / Ошибок (мини-лог).

## 📋 4. Списки: Инбаунды, Пользователи, Соединения (Data Grids)

Самая важная часть. Нужно решить проблему "сухих" таблиц, которые не читаются на телефоне. Скрещиваем функционал 3x-ui со стилем metacubexd.

- [ ] **Адаптивные таблицы (Responsive Tables)**
  - **Desktop:** Классическая таблица, но в стиле карточки. Чередование цвета строк, липкий заголовок.
  - **Mobile:** Таблица скрывается, вместо нее рендерится список из *Карточек*. В карточке крупно имя/порт, статус в углу, ползунок трафика снизу.
- [ ] **Визуализация Трафика и Сроков**
  - **Traffic Bar:** Столбец трафика выглядит как прогресс-бар из 3x-ui: `(Использовано) 15 GB / 50 GB (Лимит)`. 
    - Цвет меняется: Зеленый (< 70%) -> Желтый (70-90%) -> Красный (> 90%).
  - **Сроки действия:** Дата понятным форматом (например: "Осталось 5 дней").
- [ ] **Меню действий (Actions)**
  - Вместо 4-5 кнопок в ряд — одна кнопка `[...]` (More / Actions), открывающая Dropdown.
  - Кнопки быстрого копирования ссылки (`vless://...`) и просмотра QR-кода прямо в строке таблицы.
- [ ] **Умный поиск и фильтрация**
  - Поисковая строка над таблицами (поиск по имени, порту, UUID).
  - Фильтры (Показать только активных, Показать истекающих).

## 📝 5. Формы Создания и Редактирования (Forms/UX)

Упрощаем взаимодействие. Пользователь не должен переходить на пустую страницу `/inbounds/create`.

- [ ] **Логика "Шторки" (Drawer & Modal)**
  - Заменить роуты `/inbounds/create` и `/inbounds/:id/edit` на открытие боковой панели (Slide-over) поверх списка инбаундов. Это сохраняет контекст.
- [ ] **Группировка настроек**
  - Сложные формы инбаундов разбиты на стилистические блоки в карточках (или вкладки Tabs):
    1. **Основные** (Протокол, Порт, Клиенты).
    2. **Транспорт (Stream Settings)** (TCP, WS, gRPC, Reality/TLS).
    3. **Sniffing & Routing** (Настройки перехвата и маршрутизации).
- [ ] **Валидация и обратная связь**
  - Подсветка неверных полей красным цветом с понятной подсказкой под полем (интеграция с Zod).
  - Уведомления об успехе или ошибках через `sonner` (всплывающие тосты внизу экрана).

## ✨ 6. Анимации, Полировка (Polish & Feel)

Внимание к деталям, которое отличает премиум-продукт от базовой поделки.

- [ ] **Hover & Active states:** Все интерактивные элементы реагируют на курсор и нажатие (изменение цвета `hover:bg-white/10`, масштабирование `active:scale-95`).
- [ ] **Скелетоны:** При ожидании ответа от Backend (например, получении списка юзеров), показывать серые мигающие блоки в форме будущей таблицы.
- [ ] **Пустые состояния (Empty States):** Если инбаундов нет, показывать красивую иконку, текст ("Вы еще не создали ни одного подключения") и большую призывную кнопку "Создать".
- [ ] **Мягкие тени и градиенты:** Использование `shadow-xl`, `ring-1 ring-white/10`, и легких фоновых `bg-gradient-to-r` для выделения важных статусов.

## 🚀 Порядок разработки (План действий для AI)

1. **Этап 1: Подготовка CSS & Дизайн-системы**
   - Модификация `tailwind.config.js`
   - Создание общих компонентов (Card, Button, Badge, Switch).
2. **Этап 2: Layout & Навигация**
   - Переписывание `Sidebar`, `Header` и `Layout` на новый адаптивный стек с поддержкой мобильного Drawer.
3. **Этап 3: Страницы списков (Inbounds / Users)**
   - Создание адаптивной таблицы с прогресс-барами трафика.
   - Разработка компонента Dropdown для действий.
4. **Этап 4: Формы и Всплывающие окна**
   - Реализация Slide-over/Modal для создания инбаундов прямо на странице списка.
5. **Этап 5: Дашборд и Метрики**
   - Создание графиков и колец нагрузки.
6. **Этап 6: Тестирование и полировка.**
