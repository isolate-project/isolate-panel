## [1.0.0] — 2026-04-02

First stable release. Covers all phases 0–14.

### Added

#### Core Management (Phase 7–8)
- Support for Xray, Sing-box, and Mihomo proxy cores
- Start / stop / restart cores via Supervisord XML-RPC
- Core status endpoint with PID, uptime, restart count
- WARP integration: register WARP device, manage routing rules (domain/IP/CIDR), apply presets
- GeoIP/GeoSite routing rules with country and category selectors
- Automatic GeoIP database update (every 7 days)

#### User Management (Phase 1–2)
- Full CRUD for proxy users (username, email, UUID, password, token)
- Traffic quota enforcement with configurable limits
- User expiry dates with automatic deactivation
- Credential regeneration (new UUID, password, subscription token)
- Bulk user assignment to inbounds

#### Inbound / Outbound Management (Phase 3)
- Protocol Schema Registry with 25+ protocol schemas (VLESS, VMess, Trojan, Shadowsocks, TUIC, Hysteria2, Hysteria, Reality, ShadowTLS, NaïveProxy, gRPC, WebSocket, HTTP/2, and more)
- Dynamic form generation based on protocol schema
- Port conflict detection (`GET /inbounds/check-port`)
- Bulk user assign/unassign per inbound

#### Subscriptions (Phase 4)
- Auto-detect subscription format based on User-Agent
- Clash YAML, Sing-box JSON, V2Ray base64, QR code formats
- Short URL generation (`/s/:code`)
- Subscription access statistics (IP, user-agent, format, latency)

#### Certificates (Phase 5)
- ACME/Let's Encrypt certificate provisioning (HTTP-01 challenge)
- Wildcard certificate support
- Manual PEM upload
- Auto-renewal, revocation, deletion
- Dropdown endpoint for UI selectors

#### Monitoring & Statistics (Phase 6)
- Per-user traffic charts (7/30-day, hourly/daily granularity)
- Aggregated traffic overview for Dashboard charts
- Top-users chart by traffic consumption
- Active connection tracking
- Disconnect user (soft) and kick user (hard, removes from core)
- Real-time Dashboard stats via WebSocket (`GET /api/ws/dashboard`)
- Fallback to REST polling when WebSocket is unavailable

#### Backup System (Phase 9)
- AES-256-GCM streaming encrypted backup creation
- Backup restore with optional force flag
- Backup download as binary stream
- Configurable cron schedule (`GET/POST /backups/schedule`)
- Configurable retention policy

#### Notifications (Phase 10)
- Telegram Bot integration
- Webhook integration with configurable URL and secret
- Test notification endpoint (`POST /notifications/test`)
- Notification history with pagination

#### CLI (Phase 11)
- Cobra-based CLI tool (`isolate-panel`) with separate `go.mod`
- Commands: `user list/create/delete`, `core start/stop/restart`, `backup create/restore`
- Output formats: table, JSON, YAML
- Shell completions: bash, zsh, fish

#### Docker Deployment (Phase 12)
- Multi-stage Dockerfile (Alpine base, non-root user `isolate`)
- `setcap cap_net_bind_service` for ports < 1024
- Docker Compose for production and development (hot reload via Air + Vite)
- `docker-healthcheck.sh` checks Supervisord process states
- Log rotation (50 MB per file, 3 backups)
- Health check endpoint: `GET /health`

#### Security (Phase 14)
- TOTP / 2FA for admin login (`/auth/totp/setup`, `/verify`, `/disable`, `/status`)
- Argon2id password hashing
- JWT access tokens (15 min) + refresh tokens (7 days, stored hashed with SHA-256)
- Rate limiting: Login 5/min, Protected 60/min, Heavy ops 10/min
- Audit log for all state-changing admin actions (user.create, user.delete, core.start, …)
- Content Security Policy + X-Frame-Options + X-XSS-Protection + Referrer-Policy headers
- Request validation via `go-playground/validator/v10` (`BindAndValidate[T]` helper)
- Super-admin role (audit log access, elevated permissions)

#### Performance & DX (Phase 14)
- Frontend code splitting — all 17 pages lazy-loaded via `preact/compat`
- Vite per-route chunks: initial bundle ~120 KB (vs ~800 KB without splitting)
- Error boundaries on all pages
- Version injected via linker flags (`-ldflags "-X version.Version=$(VERSION)"`)
- Swagger/OpenAPI docs at `GET /api/docs` (generated by swaggo/swag)
- Traffic auto-reset scheduler: weekly (Monday midnight) or monthly (1st of month)
- Database performance indexes on `subscription_token`, `traffic_used_bytes`, `is_active`

#### Developer Experience
- `golangci-lint v2` with errcheck, staticcheck, gosec, misspell, prealloc linters
- CI pipeline: Go 1.25, Node 22, test + lint + coverage
- Backend tests: unit, integration, e2e, edge-cases, goroutine leak tests
- Frontend tests: Vitest unit/component + Playwright e2e (Chromium)
- Architecture docs, Contributing guide, API reference

### Changed

- Main entry point refactored: 460-line `main.go` → modular `internal/app/` (providers, routes, background workers)
- Module path updated: `github.com/vovk4morkovk4/isolate-panel` → `github.com/isolate-project/isolate-panel`
- `docker-entrypoint.sh` reads `ADMIN_PASSWORD` from environment (no more hardcoded `admin`)
- Database schema initialized via `cmd/migrate` (no duplication in entrypoint scripts)
- Go version in CI and Dockerfile updated to 1.25

### Fixed

- `CacheManager` dead code removed from `UserService` and `ConfigService` (field stored but never used)
- TypeScript strict mode: all `any` types eliminated, ESLint `@typescript-eslint/no-explicit-any` → error
- Missing ARIA labels added to interactive UI components
- Focus management in modal dialogs

## [0.1.0] — 2026-03-01

### Added
- Project scaffolding: Go backend (Fiber v3, GORM, SQLite), Preact frontend (Vite, TypeScript, Tailwind)
- Authentication system: JWT, Argon2id, refresh tokens
- Core lifecycle management via Supervisord
- Basic user, inbound, outbound management
- SQLite database with migrations
- Docker Compose setup
